# V1.5 Windows Autostart Plan

本方案已经实施。任务名称为`Codex-PhotogalleryV1-Autostart`，当前用户登录后延迟30秒执行长期宿主`run-gallery-host.ps1`。

## Recommended trigger

首次建议使用“当前维护用户登录时”触发，而不是系统启动时：

- 触发：At log on，限定当前维护账户；
- 延迟：30秒，等待磁盘、网络和ZeroTier初始化；
- 重试：启动失败后间隔5分钟重试，最多3次；
- 互斥：运行中不启动新实例；
- 不设置固定结束时间。

原因：当前Node安装在用户配置目录，使用SYSTEM或未登录账户可能无法访问Node、用户目录和媒体盘。稳定观察后才评审“计算机启动时且无用户登录”模式。

## Action

工作目录：项目根目录。

程序：

```text
powershell.exe
```

参数模板：

```text
-NoProfile -ExecutionPolicy Bypass -File "<project-root>\scripts\run-gallery-host.ps1" -EnvFile "D:\GalleryRuntime\config\gallery.env"
```

不要复制真实Node路径到Git文档或任务导出文件中；在本机创建任务时填写。

## Permissions

- 运行身份：拥有代码读取、Runtime读写和媒体读取权限的维护用户。
- 初始阶段选择“仅用户登录时运行”，便于观察错误窗口和日志。
- 网站启动本身不需要管理员权限。
- 不把防火墙配置并入启动任务。
- 不授予媒体删除权限；保持`ALLOW_REMOTE_DELETE=0`。

## Preconditions

创建任务前必须：

1. `check-environment.ps1`通过。
2. `status-gallery.ps1`能准确区分running/stopped。
3. 手工启动和停止至少再成功一次。
4. 48102防火墙策略由用户单独确认。
5. 数据库离线备份与恢复步骤已演练。

## Disable

在任务计划程序中选择任务并点击“禁用”。禁用只阻止下次启动，不会停止当前Node进程；随后运行：

```powershell
.\scripts\stop-gallery.ps1
```

## Rollback and removal

1. 禁用计划任务。
2. 使用`stop-gallery.ps1`停止V1。
3. 保留任务配置和Runtime现场用于诊断。
4. 如果需要回到旧项目，先确认48101空闲，再使用旧入口。
5. 只有确认不再需要时才删除任务；删除任务不删除Runtime、数据库或日志。

## Failure handling

- 任务历史和`gallery.stderr.log`同时检查。
- PID存在但端口不通时，`status-gallery.ps1`应显示`degraded`。
- 禁止在失败处理动作中自动删除PID、数据库、缓存或媒体。
- 连续失败后保持停止状态并人工处理，不无限重启。

## Installed result

- User：当前维护用户；
- LogonType：Interactive；
- RunLevel：Limited；
- Delay：`PT30S`；
- WorkingDirectory：项目根目录；
- MultipleInstances：IgnoreNew；
- ExecutionTimeLimit：`PT0S`；
- 手工触发结果：`LastTaskResult=0`；
- 网站运行期间任务保持Running；手工CMD和登录触发共用该任务。
