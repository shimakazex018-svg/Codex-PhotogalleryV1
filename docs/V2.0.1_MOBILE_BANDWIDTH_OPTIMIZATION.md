# V2.0.1 移动端流量与按需加载优化

## 根因与实现

优化前缺失 thumbnail 会返回原图，首页20个轮播文件合计150.15 MiB并全部设置`src`；目录首批120、图片DOM首批50，fetch不可取消，视频可退化为`preload="metadata"`。目录API本身没有深层递归预取。

V2.0.1新增`/api/image-preview?url=...&size=768`。它只接受媒体根图片或可由SQLite解析的兼容thumbnail URL；缓存key包含规范路径、size、mtime、最大边768、WebP和质量78。生成后302到版本化`/image-previews/<sha256>.webp`，响应支持immutable、ETag、Last-Modified和304。FFmpeg异步生成，进程内单并发、同key去重；失败返回503并记录日志，绝不回退原图。

首页轮播只加载当前和下一项；首页collection limit 40、目录媒体首批40、图片DOM首批24。列表图片进入约一屏缓冲区才设置preview URL，灯箱点击后才加载原图。hash切换取消旧页面fetch并用generation拒绝旧响应；搜索取消前一请求。video始终`data-src + preload="none"`，离页暂停并释放src。

## 配置与维护

```text
ENABLE_IMAGE_THUMBNAIL_GENERATION=0
ENABLE_IMAGE_PREVIEW_GENERATION=1
IMAGE_PREVIEW_DIR=D:\GalleryRuntime\image-previews
IMAGE_PREVIEW_MAX_EDGE=768
IMAGE_PREVIEW_QUALITY=78
```

只读统计：`.\scripts\check-image-preview-cache.ps1 -PreviewDir D:\GalleryRuntime\image-previews`。本版本不自动删除、不全库扫描、不预生成；旧轮播文件也不会自动清理。

## 验证与限制

隔离smoke使用合成图片、独立SQLite/端口和临时Runtime：8个相同并发请求只生成1个文件；缓存命中不新增；mtime变化生成新key；路径穿越、非图片、缺失文件和损坏图片均受控；失败后Node存活；root limit返回40。最终2个预览共1,756 bytes，测试后已清理。

真实Chrome前后Network对比未执行：当前Native Messaging Host/manifest缺失，Codex无法连接扩展。需重新安装Chrome插件后按审计文档补录HAR。当前还没有自动淘汰、容量上限、磁盘告警或跨进程锁；文本gzip和稳定游标分页延后。

正式Runtime已配置独立`D:\GalleryRuntime\image-previews`并重启。最小验收：root返回40项、highlights返回20项；只请求首个highlight预览后生成1个33,926-byte WebP（0.03 MiB），HTTP 200、`image/webp`；版本化URL返回immutable缓存头，匹配ETag的条件请求返回304。未执行批量生成。正式状态仍因既有任务计划程序缺失和ListenerPID不可读显示degraded，但HTTP 200且Node/Host父子关系正确。
