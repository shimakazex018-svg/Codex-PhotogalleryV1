# V1.4.2 Runtime Implementation

实施日期：2026-07-12

## Scope and result

V1.4.2 已建立独立于代码仓库和旧项目的运行目录、外部配置、数据库副本及 PowerShell 运维入口。本阶段没有启动网站、打开数据库、删除旧项目、修改原始媒体或修改业务代码。

## Created runtime structure

```text
D:\GalleryRuntime\
├─ config\
│  ├─ gallery.env
│  └─ gallery-db-migration.sha256
├─ data\
│  ├─ gallery.db
│  ├─ thumbnails\ -> D:\GalleryRuntime\thumbnails
│  └─ logs\       -> D:\GalleryRuntime\logs
├─ thumbnails\
├─ video-posters\
├─ hls\
├─ logs\
└─ trash\
```

`data\thumbnails` 和 `data\logs` 是 Windows directory junction。原因是当前服务端固定从 `DATA_DIR` 派生图片缩略图和日志目录，而 V1.4.2 又要求这些内容位于 runtime 顶层；目录联接在不修改业务逻辑的前提下兼容两者。

## Configuration

真实配置位于 `D:\GalleryRuntime\config\gallery.env`，包含用户指定的 11 个键：

- `PORT=48102`
- `DATA_DIR`
- `PHOTOS_DIR`
- `THUMBNAIL_DIR`
- `POSTER_DIR`
- `HLS_DIR`
- `TRASH_DIR`
- `FFMPEG_PATH`
- `FFPROBE_PATH`
- `ALLOW_REMOTE_DELETE=0`

该文件位于 Git 仓库外，不提交。配置解析器只接受白名单 `KEY=VALUE`，不执行 PowerShell 表达式。

兼容映射：当前 `server.js` 使用 `THUMBNAILS_DIR` 表示视频 poster 目录，因此启动器把 `POSTER_DIR` 注入为 `THUMBNAILS_DIR`。当前代码不读取 `THUMBNAIL_DIR`，图片缩略图通过上述 directory junction 落到顶层目录。

## Scripts

| 文件 | 作用 |
|---|---|
| `scripts/gallery-runtime-common.ps1` | 安全解析 env、路径/工具/端口/数据库检查、环境变量注入 |
| `scripts/check-environment.ps1` | 只执行预检，不启动网站 |
| `scripts/start-gallery.ps1` | 预检后启动新服务，日志和 JSON PID 元数据写入 runtime |
| `scripts/stop-gallery.ps1` | 核对 PID、Node 路径和启动时间后只停止 V1 进程 |
| `scripts/configure-firewall-48102.ps1` | 显式创建 Private profile 的 TCP 48102 入站规则 |

防火墙脚本没有执行。启动脚本不会自动修改系统防火墙。

## Startup method

先在不启动网站的情况下检查：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-environment.ps1 `
  -EnvFile D:\GalleryRuntime\config\gallery.env `
  -NodePath <node-exe>
```

用户明确批准首次启动后才可执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\start-gallery.ps1 `
  -EnvFile D:\GalleryRuntime\config\gallery.env `
  -NodePath <node-exe>
```

停止：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\stop-gallery.ps1
```

`NODE_EXE` 可作为启动器进程环境变量提供，也可使用 `-NodePath`；它不写入用户指定的 `gallery.env`。

## Rollback

本阶段旧项目、旧数据库和端口 48101 均未修改。回滚只需：

1. 如果未来已启动 V1，先运行 `stop-gallery.ps1`。
2. 保留 `D:\GalleryRuntime` 现场，不删除或覆盖。
3. 确认没有媒体移动任务运行；V1.4.2 当前强制远程删除关闭。
4. 使用旧项目原入口和旧数据库恢复端口 48101。
5. 验证旧首页、数据库统计、搜索、图片和视频。

## Remaining risks and next gate

- 本阶段没有打开目标 SQLite，因此数据逻辑完整性和业务统计尚未验证。
- 网站没有启动，48102 的 HTTP、API、图片、视频及日志写入尚未验收。
- 视频 poster 源路径映射 404 仍是已知问题，本阶段禁止修复。
- 顶层 `trash` 与媒体目录不在同一磁盘；跨卷移动风险需在隔离测试中确认，且当前保持 `ALLOW_REMOTE_DELETE=0`。
- 防火墙规则尚未创建；是否开放局域网访问需用户单独决定。
- 旧项目在首次运行验收、备份和回滚演练完成前不能删除。

下一阶段只能在用户明确批准后进行 V1.4.3 首次只读启动与验收。
