# V1.5 Operation Manual

## Start

普通用户推荐直接双击项目根目录的`Start Gallery.cmd`。它从`%~dp0`定位项目、调用现有PowerShell启动器、显示PID/URL，并在成功后打开默认浏览器。

启动CMD只触发任务计划程序；任务中的`run-gallery-host.ps1`在Node存活期间保持运行。因此CMD窗口可安全关闭。

在项目根目录运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-environment.ps1 `
  -EnvFile D:\GalleryRuntime\config\gallery.env `
  -NodePath <node-exe>

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\start-gallery.ps1 `
  -EnvFile D:\GalleryRuntime\config\gallery.env `
  -NodePath <node-exe>
```

启动脚本会拒绝重复PID或占用端口。不要使用旧48101脚本作为日常入口。

## Stop

可直接双击项目根目录的`Stop Gallery.cmd`。

```powershell
.\scripts\stop-gallery.ps1
```

脚本核对PID元数据、Node路径和启动时间后停止进程，不按进程名批量终止Node。

## Status

可直接双击项目根目录的`Gallery Status.cmd`，窗口会等待按键以便查看结果。

```powershell
.\scripts\status-gallery.ps1
```

输出：状态、PID、48102连接状态、Node状态和路径、Runtime、env、stdout与stderr日志位置。

## Logs

```text
D:\GalleryRuntime\logs\gallery.stdout.log
D:\GalleryRuntime\logs\gallery.stderr.log
```

先看stderr，再看stdout和status。日志不得提交Git。

## Update code

1. 运行`status-gallery.ps1`记录当前状态。
2. 运行`stop-gallery.ps1`。
3. 执行`git status --short`；有未知修改时停止，不覆盖。
4. 获取远程前先检查分支和远程历史；禁止force pull/push。
5. 更新后执行四个核心JS的`node --check`和PowerShell脚本解析。
6. 阅读CHANGELOG和HANDOFF。
7. 执行环境预检，再启动并验证首页和核心只读API。

## Runtime backup

Runtime包含数据库、配置、日志和可重建缓存。备份前停止服务；优先备份：

1. `config\gallery.env`，存入受控私有备份，不能进入Git；
2. `data\gallery.db`；
3. 缓存状态和必要日志；
4. poster/HLS可按容量策略选择备份，图片thumbnail可重建且当前生成关闭。

不要把Runtime备份放进项目目录。

## Database backup

1. 停止网站并确认`status=stopped`。
2. 确认没有`gallery.db-wal`或`gallery.db-shm`。
3. 复制到Runtime外的版本化备份目录，文件名包含UTC时间。
4. 记录源和备份SHA256、大小和时间。
5. 只在副本上做完整性验证；不要用应用代码打开源库冒充只读检查。

当前迁移基线SHA256记录在V1.4文档中；日常数据库发生收藏/最近浏览等合法写入后，哈希变化是正常的，备份时应记录新的值。

## Failure recovery

### Process not running

- 查看`gallery.stderr.log`。
- 运行`check-environment.ps1`。
- 检查Node、Runtime、数据库、媒体盘和48102。
- 不通过手工`node server.js`绕过正式配置。

### PID exists but port unavailable

- 运行`status-gallery.ps1`确认`degraded`。
- 核对PID是否为当前项目Node。
- 使用`stop-gallery.ps1`，不要批量结束所有Node。
- 保存日志后再启动。

### Database failure

- 立即停止V1，不运行扫描或自动修复。
- 保留故障库、WAL/SHM和日志现场。
- 从最近验证过的离线副本恢复到新路径，先核对哈希和统计。
- 不覆盖旧项目数据库或原始媒体。

### Media unavailable

- 确认媒体盘和`PHOTOS_DIR`存在、可读。
- 不触发大规模扫描。
- 路径恢复后先抽样验证原图和视频Range。

## LAN and ZeroTier

当前本机地址见`docs/V1.5_ACCEPTANCE_REPORT.md`。防火墙规则需管理员显式运行网络脚本，网站启动脚本不会自动修改防火墙。

实体手机验收至少检查：首页、目录、图片、视频播放、搜索、收藏、最近浏览和设置页。完成前不要声称移动端已完全验收。

## Autostart

双击`Install Autostart.cmd`安装当前用户登录任务；双击`Uninstall Autostart.cmd`只删除该任务，不停止当前网站，也不删除项目、Runtime或日志。

当前任务：

```text
Codex-PhotogalleryV1-Autostart
```

任务使用Limited权限、登录后延迟30秒、IgnoreNew重复策略。网站已运行时任务再次触发不会创建第二个Node进程。

LAN访问需要管理员双击`Configure LAN Access.cmd`创建严格范围的48102规则。该入口只提升防火墙脚本，不提升网站。
