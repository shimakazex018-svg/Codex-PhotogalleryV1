# V1.5 Browser Access Diagnosis

诊断日期：2026-07-12

## Scope

本次只诊断“内部HTTP测试正常但真实Windows浏览器无响应”。没有处理自动启动、UI、性能、HLS、缓存、GitHub、防火墙或ZeroTier，也没有修改业务代码、端口、数据库或媒体。

## Confirmed root cause

首次检查时网站实际已经停止：

- `D:\GalleryRuntime\logs\gallery.pid`仍记录旧PID `56500`；
- PID `56500`对应进程不存在；
- `Get-NetTCPConnection`没有返回监听项，并因当前令牌权限提示Access denied；
- `netstat -ano | findstr :48102`没有任何输出；
- `status-gallery.ps1`返回`stopped`。

因此用户浏览器无响应的直接原因是48102当时没有服务监听，而不是HTML、HTTPS重定向、MIME或Windows代理。

旧stdout只记录了正常启动，没有退出原因；stderr为空。现有日志无法确定PID `56500`为什么退出。可能的外部进程生命周期、用户会话或宿主清理仍需后续观察，但本阶段没有证据授权修改启动架构。

## Manual browser checklist

请用户在Windows真实浏览器地址栏中逐个完整输入，必须保留`http://`：

```text
http://127.0.0.1:48102/
http://localhost:48102/
http://192.168.31.153:48102/
```

优先从`127.0.0.1`开始。若浏览器自动改为`https://`，不要继续访问错误HTTPS地址，记录地址栏最终内容和浏览器的“始终使用安全连接/HTTPS-First”状态。

每个地址请记录以下一种具体表现：

- `ERR_CONNECTION_REFUSED`；
- `ERR_CONNECTION_RESET`；
- `ERR_TIMED_OUT`；
- 自动跳转HTTPS；
- 页面空白；
- 一直转圈；
- 页面打开但图片不显示；
- 其他错误，请保存完整错误文字和截图。

如果`127.0.0.1`仍失败，请立即在另一个PowerShell窗口运行：

```powershell
.\scripts\status-gallery.ps1
netstat -ano | findstr :48102
```

若状态再次变为`stopped`，说明问题是进程未持续运行，需要继续调查退出生命周期，而不是修改网页或浏览器代理。
