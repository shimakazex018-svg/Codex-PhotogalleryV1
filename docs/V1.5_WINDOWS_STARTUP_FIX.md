# V1.5 Windows Startup and LAN Access Fix

实施日期：2026-07-12

## Problem summary

本阶段只处理：

1. 用户关闭`Start Gallery.cmd`窗口后网站退出；
2. 服务器本机可访问48102，但同一LAN远程设备无响应。

没有修改业务JavaScript、HTML/CSS、数据库、Runtime结构、媒体路径、端口、缓存、HLS、ZeroTier或Clash。

## Previous lifecycle diagnosis

旧结构：

```text
Start Gallery.cmd
  -> short-lived start-gallery.ps1
       -> Start-Process node server.js
  -> PowerShell exits
  -> Scheduled Task returns Ready
  -> Node has no persistent task host
```

复现记录：

- CMD PID：`1488`；
- 启动PowerShell PID：`7824`，在采样前已退出；
- Node PID：`15752`；
- Node记录的父PID：`7824`；
- 旧计划任务状态：Ready，而Node仍在运行。

隐藏CMD强制关闭的自动化等价测试中，Node在12秒后仍存活，因此本环境没有捕获用户桌面上的具体`CTRL_CLOSE_EVENT`。不能声称已证明某一个Windows控制信号。

已确认的架构根因是：旧任务只运行短生命周期启动器，Node成为无持续宿主的派生进程；任务计划程序不再处于Running状态，无法承担Node生命周期。用户桌面关闭窗口、会话或宿主清理时，网站没有独立托管边界。

## New task-hosted architecture

```text
Windows logon or Start Gallery.cmd
  -> Start-ScheduledTask: Codex-PhotogalleryV1-Autostart
  -> Task Scheduler starts run-gallery-host.ps1
  -> host parses gallery.env and injects environment
  -> host starts one Node process and waits for it
  -> task remains Running for the website lifetime
```

`run-gallery-host.ps1`：

- 读取`D:\GalleryRuntime\config\gallery.env`；
- 复用`gallery-runtime-common.ps1`进行全部校验和环境注入；
- 启动一个Node并在其运行期间持续等待；
- stdout/stderr继续写入Runtime现有日志；
- PID元数据增加host PID、父PID和任务名；
- host日志记录启动、Node PID、退出时间和退出码；
- Node退出后任务结束，不无限重启。

任务定义：

- Trigger：当前用户登录；
- Delay：`PT30S`；
- Action：完整路径`run-gallery-host.ps1`；
- WorkingDirectory：项目根；
- RunLevel：Limited；
- MultipleInstances：IgnoreNew；
- ExecutionTimeLimit：`PT0S`，不设置三天默认终止限制。

## Start Gallery.cmd

CMD只调用`start-gallery.ps1`：

1. 读取status；
2. 已运行则显示PID和URL；
3. 未运行则`Start-ScheduledTask`；
4. 最多等待15秒，要求任务Running、Node存在、48102监听且首页HTTP 200；
5. 成功后CMD可关闭，网站由任务host继续托管。

## Stop Gallery.cmd

CMD只调用`stop-gallery.ps1`。PowerShell脚本：

1. 读取并验证同一JSON PID元数据；
2. 只停止记录的gallery Node PID；
3. 等待host/任务退出，必要时停止同名任务实例；
4. 保留任务定义，供下次登录或手工启动；
5. 不按名称批量结束Node。

## LAN diagnosis

- LAN接口：活动以太网；
- IPv4：`192.168.31.153/24`；
- 默认网关：`192.168.31.1`；
- Windows网络类别：Private；
- Gallery监听：`0.0.0.0:48102`；
- 监听PID：与当前gallery Node一致；

- 初始防火墙规则：不存在。

本机LAN URL正常而远端无响应，确认Windows入站防火墙规则缺失是远程访问阻断点。

## Firewall design

规则名称：

```text
Codex-PhotogalleryV1-48102-LAN
```

当前Private LAN的目标规则：

- Direction：Inbound；
- Action：Allow；
- Protocol：TCP；
- LocalPort：48102；
- LocalAddress：`192.168.31.153`；
- RemoteAddress：`LocalSubnet`；
- Profile：Private；
- Enabled：True。

脚本不会修改ZeroTier。如果LAN以后是Public，脚本只允许`192.168.31.0/24`，不会使用RemoteAddress=Any。

`Configure LAN Access.cmd`只提升防火墙配置PowerShell。网站和任务继续使用Limited权限。

### Current application status

自动化会话触发UAC时操作被取消；当前进程不是管理员，最终确认规则尚未创建。需要用户在Windows桌面双击`Configure LAN Access.cmd`并批准UAC，然后重新读取规则。

因此：服务器本机LAN访问已通过；跨设备访问和防火墙应用尚未完成，不能声称实体设备已验收。

防火墙规则创建后，在同一局域网设备打开：

```text

```

确认：

1. 手机关闭移动数据或明确使用同一Wi-Fi；
2. 手机地址属于`192.168.31.0/24`；
3. 浏览器使用HTTP，不是HTTPS；
4. 首页、图片目录、原图和视频播放正常。

## Troubleshooting

远端仍失败时依次检查：

1. `Gallery Status.cmd`中任务、PID、Listening和HttpStatus；
2. `netstat -ano | findstr :48102`是否为`0.0.0.0`且PID匹配；
3. LAN IP是否仍为`192.168.31.153/24`；
4. LAN网络类别是否仍为Private；
5. 防火墙规则的Profile、LocalAddress、RemoteAddress和LocalPort；
6. 客户端是否同网段且未自动切换HTTPS。

## 2026-07-13 hidden-host correction

### What the user-visible long-lived window was

再次严格复现时，旧任务定义为`LogonType=Interactive`、`Hidden=False`，Action直接执行：

```text
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\A8 Codex\Codex-PhotogalleryV1\scripts\run-gallery-host.ps1" -EnvFile "D:\GalleryRuntime\config\gallery.env"
```

实测进程树：

```text
Task Scheduler (parent PID 1124)
└─ powershell.exe run-gallery-host.ps1 (Host PID 17412)
   └─ node.exe server.js (Node PID 17144)
```

另有瞬时`Start Gallery.cmd`进程PID `12700`。长期存在并可能显示控制台的是任务创建的Host PowerShell，而不是瞬时启动CMD，也不是Node。关闭Host会使任务立即从Running变为Ready；自动化使用`Stop-Process`终止Host时Node暂时成为孤儿并继续监听，这与用户桌面关闭控制台时整组退出的`CTRL_CLOSE_EVENT`行为不同，因此不虚构精确信号链。确定的根因是任务Action允许长期交互式可见控制台，用户可关闭该唯一宿主边界。

历史PID `16400`、`13248`、`17680`来自不同启动轮次：`13248`曾是Host/Node父PID，`17680`曾是Node/监听PID，`16400`在另一轮验收中是Node PID。PID元数据字段没有把Host与Node写反，但旧status字段名`PID`不够直观。

### Final hidden Action

任务Action现为：

```text
C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -NoProfile -NonInteractive -WindowStyle Hidden -ExecutionPolicy Bypass -File "D:\A8 Codex\Codex-PhotogalleryV1\scripts\run-gallery-host.ps1" -EnvFile "D:\GalleryRuntime\config\gallery.env"
```

`run-gallery-host.ps1`启动Node时也显式使用`-WindowStyle Hidden`。未引入VBS、第三方程序、服务或自动重启。

`status-gallery.ps1`现在分别输出`ScheduledTaskState`、`HostPID`、`NodePID`、`NodeParentPID`、`ListenerPID`和`HttpStatus`。只有Node父PID等于Host PID且监听PID等于Node PID时才能显示running；否则显示degraded。
